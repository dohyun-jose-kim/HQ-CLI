# 프로젝트 목표 & 버전 이력

---

## 목표

일반 소비자가 자신의 영양제 섭취량을 입력하면, 과잉 섭취 위험도(HQ)를 정량적으로 확인할 수 있는 도구를 만드는 것이 이 프로젝트의 목표다.

현재 버전은 **CLI 환경에서 구동 가능한 프로그램을 완성하는 것**에 초점을 맞추었다. 장기적으로는 일반 사용자가 접근 가능한 웹 서비스 형태로의 확장을 지향한다.

---

## 버전별 목표 갱신 내역

### v0 — CLI 원본 (`CLI/`)

초기 구현. 기본 파이프라인(입력 → 파싱 → HQ 계산 → 시각화)을 구축하여 동작하는 프로토타입을 완성했다.

- 4단계 파이프라인 구조 확립 (initial_reaction → parsing → data_processor → result_maker)
- DEV_MODE = True (개발 모드 기본)
- 파싱 실패 시 해당 항목을 조용히 건너뜀
- 문서/데이터 폴더 미정비

### v1 — HQ-CLI_byDH (`HQ-CLI_byDH/`)

v0의 구조를 유지하면서 정확도와 UX를 개선하고, 배포 가능한 형태로 정비했다.

**정확도 개선**
- fuzzy 매핑 threshold 강화: difflib cutoff 0.55 → 0.70
- ML threshold 강화: TF-IDF 코사인 유사도 0.25 → 0.45
- "철판"→"철분" 같은 거짓양성 차단

**UX 개선**
- 파싱 실패 시 재입력 기능 추가 (성별/나이 유지, 영양소만 재입력)
- 단위 누락 감지 및 경고
- `q` 입력으로 어느 단계에서든 즉시 종료
- `_input_or_quit()` 헬퍼 추출로 종료 로직 통일

**배포 정비**
- DEV_MODE = False (배포 기본값)
- README.md 추가
- 의존성 파일 정규화 (`requirements.txt`)
- 데이터/에셋 폴더 구조 정리

### 향후 — 개선 방향

현재 버전의 한계를 인지하고 있으며, 다음 방향으로 개선을 검토한다.

1. **입력 방식 개선**: 현재는 interactive하게 한 줄씩 수동 입력하는 방식이다. 개인화된 입력을 파일(예: CSV, JSON)로 받아 일괄 처리할 수 있도록 개선이 필요하다.
2. **출력 분석 강화**: 현재는 구동 가능한 프로그램을 만드는 것이 목표였기에, 출력 데이터 분석이 최소한에 머물러 있다. HQ 결과에 대한 더 풍부한 해석과 시각화를 제공할 필요가 있다.
3. **사용자 접근성**: CLI 환경은 일반 소비자에게 적절하지 않다. HTML/CSS 등을 활용한 웹 인터페이스로 전환하여 접근성을 높이는 것을 지향한다.
