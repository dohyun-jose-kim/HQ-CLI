## 알려진 한계

### 데이터 관련

- **영양소 15종 제한**: UL 데이터에는 20종이 있으나, 이 중 15종만 지원한다. Nicotinic acid, Nicotinamide, Folate, Fluorine, Magnesium은 제외되어 있다.
- **MHI 데이터 열 불일치**: Folate, Fluorine, Magnesium은 UL에는 있지만 MHI에는 없어 HQ 계산이 불가하다.
- **UL/MHI 데이터 수동 관리**: 데이터가 자동으로 업데이트되지 않는다. 국민건강영양조사 데이터가 갱신되면 수동으로 전처리하여 교체해야 한다.
- **개인 식이 편차 미반영**: MHI는 연령/성별 그룹의 평균값이다. 개인의 실제 식이 섭취량과 차이가 클 수 있다.

### 입력/UX 관련

- **수동 입력 방식**: interactive하게 한 줄씩 수동 입력해야 한다. 개인화된 입력을 파일(CSV, JSON 등)로 받아 일괄 처리하는 기능이 없다.
- **CLI 환경의 접근성**: 일반 소비자가 사용하기에 CLI 환경은 적절하지 않다. 웹 인터페이스(HTML/CSS 등)가 필요하다.

### 출력/분석 관련

- **출력 분석 최소한**: 현재 버전은 구동 가능한 프로그램을 만드는 것이 목표였기에, HQ 결과에 대한 해석과 시각화가 최소한에 머물러 있다.

### 파싱 관련

- **ML fallback 한계**: threshold를 강화(v1: 0.70, v2: 0.45)했으나, 여전히 유사한 문자열에 대한 오매핑 가능성이 존재한다.

## 개선 가능 영역

| 영역 | 현재 상태 | 개선 방향 |
|------|-----------|-----------|
| 입력 방식 | 수동 1건씩 입력 | 파일 기반 일괄 입력 (CSV, JSON) |
| 출력 분석 | HQ 값 + 막대 그래프 | 더 풍부한 해석, 비교 분석, 추세 시각화 |
| 사용자 접근성 | CLI | 웹 인터페이스 (HTML/CSS) |
| 데이터 관리 | 수동 교체 | 자동/반자동 업데이트 파이프라인 |
| 파싱 정확도 | 딕셔너리 + ML fallback | 테스트 커버리지 확대, 별칭 사전 보강 |

## 팀 논의 제안

### 분석 방식 재검토 필요

`DataAnalysis_vf.ipynb`의 분석에서 사용한 loop 기반 반복 분석 방식을 이 CLI에도 적용하려 했으나, 기존 방식에 문제가 있음을 확인했다. 이 프로젝트에서 누적 결과를 분석할 때는 별도의 분석 방식을 새로 설계해야 한다.

> **Note**: 구체적인 분석 방식은 팀 논의를 통해 결정할 것.
