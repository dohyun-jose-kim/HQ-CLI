# 요구사항 & 기능 상세

---

## 배경 & 목적

### 어떤 문제를 해결하는가

건강기능식품 시장 성장으로 영양제를 일상적으로 섭취하는 인구가 늘고 있으나, 과잉 섭취 위험에 대한 인식은 부족하다. 특히 여러 제품을 병용할 경우 동일 영양소가 중복되어 상한섭취량(UL)을 초과할 수 있다. 이 스크립트는 사용자의 영양제 섭취량을 받아 위험지수(HQ)를 계산하고, 과잉 여부를 정량적으로 판단해준다.

### 누가, 어떤 상황에서 사용하는가

- **대상**: 영양제를 섭취하는 일반 소비자
- **상황**: 자신이 복용 중인 영양제의 섭취량이 안전 범위 내인지 확인하고 싶을 때
- **환경**: CLI(터미널) 환경에서 `python main.py`로 실행

## 기능 요구사항 & Capabilities

### 사용자 입력 수집 — `initial_reaction.py`

**요구**: 성별, 연령, 영양소 섭취량을 순차적으로 수집한다.

**구현**:
- 성별, 출생 연도를 순차적으로 입력받는다.
- 영양소명과 섭취량을 `영양소: 용량단위` 형식으로 쉼표 구분하여 한 줄에 입력받는다.
- `--help`로 상세 안내, `--vv`로 지원 영양소 목록을 표시한다.
- 어느 단계에서든 `q` 입력으로 즉시 종료할 수 있다 (`_input_or_quit()` 헬퍼).

### 입력 파싱 — `parsing.py`, `parsing_v1.py`, `parsing_v2.py`

**요구**: 한글, 영문, 원소기호 등 다양한 영양소 표현을 표준 컬럼명으로 매핑한다.

**구현**:
- **1차 — 딕셔너리 매핑** (`parsing.py`): 하드코딩된 별칭 테이블로 즉시 매핑 시도.
- **2차 — ML fallback** (딕셔너리 매핑 실패 시):
  - **v1** (`parsing_v1.py`): `difflib` SequenceMatcher 기반 fuzzy 매핑 (cutoff 0.70)
  - **v2** (`parsing_v2.py`): `scikit-learn` TF-IDF 코사인 유사도 (threshold 0.45)
- `main.py`의 `DEV_MODE` 플래그로 v1/v2를 선택할 수 있으며, 배포 시에는 `DEPLOY_VERSION`으로 고정한다.
- 단위 누락 시 경고를 출력하고 사용자에게 진행/재입력 선택을 제공한다.
- 파싱 실패 시 `r` 입력으로 영양소 부분만 재입력할 수 있다 (성별/나이 유지).

### HQ 계산 — `data_processor.py`

**요구**: 사용자 섭취량과 식이 평균(MHI)을 합산하여 상한(UL) 대비 위험지수를 산출한다.

**구현**:
- 수식: `HQ = (MHI + 보충제 섭취량) / UL`
- 성별, 연령범주에 맞는 UL/MHI 행을 조회하여 영양소별 HQ를 계산한다.
- UL 또는 MHI 값이 없는 영양소는 건너뛰고 사유를 출력한다.
- 임신부/수유부 조건을 지원한다.
- Excel 파일 I/O를 모듈 수준 캐시(`_DF_CACHE`)로 최소화하여, 반복 실행 시 파일을 다시 읽지 않는다.

### 시각화 — `result_maker.py`

**요구**: HQ 결과를 직관적인 그래프로 저장한다.

**구현**:
- `matplotlib`로 영양소별 HQ 막대 그래프를 생성한다.
- HQ >= 1인 영양소를 시각적으로 구분하여 표시한다.
- `results/HQ_(timestamp).png`로 실행마다 개별 저장한다.

### 누적 결과 저장 — `main.py` (`save_result_to_excel`)

**요구**: 과거 실행 결과를 누적하여 추적 가능하게 한다.

**구현**:
- 실행마다 HQ 결과를 `results/result_accumulation.xlsx`에 1행씩 누적한다.
- 컬럼: timestamp, 성별, 나이, 연령범주, 15개 영양소 HQ 값
- `openpyxl`로 기존 파일에 행을 추가하는 방식 (파일 없으면 자동 생성).

### 오류 보고서 생성 — `main.py` (`write_error_report`)

**요구**: 사용자가 결과에 의문이 있을 때 피드백을 남길 수 있게 한다.

**구현**:
- 원본 입력, 파싱 결과, 사용자 라벨/메모를 `results/report_accumulation.csv`에 누적 저장한다.
- 에러 유형 자동 분류: 파싱_전체실패 / 파싱_일부누락 / 결과_의심
- CSV 포맷 — 개발자/관리자용 경량 누적 형식.

### 종료/재시작 흐름 — `main.py`

**요구**: 결과 확인 후 사용자가 다음 동작을 선택할 수 있다.

**구현**:
- 결과 출력 후: 종료 / 새 데이터로 재시작 / 오류 보고서 작성 중 선택.
- 오류 보고서 작성 후: 종료 / 재시작 선택.
- 재시작 시 `while True` 루프 처음으로 돌아가며, 캐시된 데이터는 유지된다.

## 비기능 요구사항

### 실행 환경

- Python 3.10 이상

### 의존성

| 패키지 | 용도 |
|--------|------|
| `pandas` | 데이터 로드 및 행 조회 |
| `openpyxl` | Excel 읽기/쓰기 (누적 결과 저장) |
| `scikit-learn` | TF-IDF 코사인 유사도 파싱 (v2) |
| `matplotlib` | HQ 막대 그래프 시각화 |

### 데이터 포맷

- 입력 데이터: `.xlsx` (Excel) — 성별/연령별 행, 영양소별 열
- 누적 결과: `.xlsx` — 사용자 친화적 포맷
- 오류 보고서: `.csv` — 개발자/관리자용 경량 누적 포맷

### 성능

- Excel 파일 I/O를 모듈 수준 캐시(`_DF_CACHE`)로 최소화하여, 반복 실행(재시작) 시 파일을 다시 읽지 않는다.

### 보안

- 해당 없음 (로컬 실행, 네트워크 미사용, 사용자 인증 없음)
